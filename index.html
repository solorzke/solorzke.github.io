Steam Theater - Web Application

Github Repo: @alex-kronish

This collaborative Senior project was mine to submit at the end of the Fall 2019 semester as I was on course to graduate. The goal was to first, establish a four server-cluster consisting of: front-end, back-end, DMZ, and a messaging broker (RabbitMq) wired into a local network router to deploy a real-world web-service product that identified the user's least-played video-games on their Steam backlog, and present Twitch/YouTube streams of those video-games.

The user can also view their friend's least-played video-games and compare/contrast what their interests are in their games. Then, scale the project to 12 servers culminating into a 3 tier-cluster of production, development, and staging platforms before deploying.

Our servers were created through Virtual Box Vm's in Ubuntu 18.04 and we had to configure our static IP addresses to the router. I was in charge of the front-end server, consisting of designing the site's interface authentication process, and writing the scripts required to produce/consume RabbitMq messages in the virtual-host's queue.

Since our product required access to the user's Steam account including searches in Twitch/YouTube, I had to parse every JSON response from its API (from RabbitMq) and use the appropriate information required for use in the site. The complexity of the project meant that collaborating with others was crucial, and together, by adopting the SCRUM workflow, we were able to build our project well and efficiently.

Hiking in Augmented Reality - UX/UI Research

Google Drive: UX/UI Design Report

I was involved working with the UX/UI research team responsible for designing a prototype of an Augmented Reality application that assists hikers in their journey using real-time updates and peripheral equipment such as Go-Pro devices. I assisted in understanding what our vision is, and its motivating problem to solve: an app that provides up-to-date information about the trail as you hike in it. The main objective was to make a mockup UI presentation (using Axure RP 8) of the application that the development team would use before commencing work on it. Each feature had to serve a purpose from our journey maps to fulfill the goals that the app must deliver in order to satisfy the growing problem that other applications aren't solving.

I identified our user stakeholders, before defining the typical user persona that would ideally use this application in 3 main different scenarios. In order to come up with that data, I conducted interviews with various subjects about their thoughts towards the potential existence of this app and gather their suggestions and feelings.

Afterwards, assist in creating our risk analysis scenarios to identify potential problems occurring with a user's usage of the app and conduct key-path situations analyses, all with the goal of collecting as much research data as possible. The amount of data collected helped us create the best possible prototype we could build, which was also used in focus-group testing with real hikers to gather their feedback about its functionality, and most importantly, their user-experience.

XLSX Report Generator

Github Repo: @solorzke

The XLSX Report Generator was a Node.js/Electron project for a client who wanted a quicker, more efficient way to produce spreadsheet reports. The problem was that the spreadsheet data pile was enormous to search through and took considerable time to produce a filtered report. Using dependencies found on NPM, the script converted the spreadsheet data into json objects (each object covered a row of cells), and ran queries to locate the relevant data that was required. Afterwards, the script generates a new XLSX file and writes the new filtered data to it. Due to limitations involving the File System access from a browser, the Node.js project had to migrate to a framework called Electron that allowed node modules to have read/write access in the file system, as a cross-platform desktop application.
